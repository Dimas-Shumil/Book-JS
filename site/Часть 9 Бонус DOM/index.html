В прошлом модуле вы познакомились с асинхронным программированием. Научились делать запросы на сторонний API, создали свои первые промисы и отловили первые события на DOM-элементах.

А также вы самостоятельно сделали галерею с картинками и попрактиковались в добавлении элементов в разметку.

Хватит ли этих знаний для создания конкурентоспособных сайтов? Пока нет.

Давайте представим, что вы фронтенд-разработчик сайта про кино. В команду прилетела задача: создать выпадающее окно поиска.

Функции такие:

1. Когда пользователь что-то нажимает на поле ввода — появляется окно.
2. Параллельно в окне появляются подсказки: фильмы, актёры и режиссёры. Они основываются на том, что вводит пользователь и меняются с каждой буквой.
3. Если пользователь кликнет вне этого окошка — оно закроется.

Пока что с имеющимися знаниями реализовать эту фичу у вас не получится. Тут нужно уметь работать с addEventListener. Обрабатывать ситуации, когда пользователь вводит что-то в инпуты или кликает мышкой в разных частях страницы.

Или другой пример. Вы — часть команды сервиса по поиску и аренде жилья. На ближайшую неделю вы должны создать меню для сайта, которое появляется по нажатии на кнопку.

Это меню:

1. Появляется по клику кнопки.
2. Содержит ссылки, которые ведут на авторизацию, регистрацию, страницу с помощью и так далее.
3. Исчезает по нажатии на любое другое место.

Здесь похожая история. Выполнить это задание можно только в том случае, если вы освоили слушатели событий и разобрались, какое событие отвечает за отправку данных из формы.

Наконец, третий пример. Вы блестяще прошли курс и пригласил вас на работу. Теперь вы — часть команды разработки.

Текущая задача: создать форму, по которой потенциальные ученики смогут оставить свои контакты. По этим контактам менеджеры свяжутся с клиентами и предложат курсы.

Вот параметры формы, которую вы должны реализовать:

1. Каждое поле проходит валидацию, когда нажимается кнопка «Отправить» или кнопка Enter на клавиатуре.
2. Если одно из полей эту валидацию не проходит — данные не отправляются. Пользователь видит, какие из полей нужно заполнить корректно.
3. Если все поля правильные, то данные отправляются, а пользователь получает оповещение об этом.

В этом случае вы, как разработчик, должны уметь получать значение из полей ввода, валидировать его и разрешать или не разрешать отправку данных.

Плохая новость: пока что у вас нет тех знаний, благодаря которым эти фичи будут внедрены в проект.

Хорошая новость: они появятся после того, как вы пройдёте этот модуль.

В нём вы:

• Научитесь доставать элементы из разметки разными способами. Например, с помощью getElementById, getElementsByClassName и так далее. Это позволит управлять любым элементом из JS и менять его свойства, не изменяя сам HTML или CSS.
• Попрактикуетесь в создании и удалении элементов из разметки, чтобы добавлять сайту динамику. Динамический сайт может обновлять контент на странице без необходимости полной перезагрузки. Это необходимо для работы с быстро меняющейся информацией. Например, с актуальными новостями, индексом ценных бумаг, статусом заказов.
• Узнаете, как манипулировать стилями элементов из JS. Изменение стилей элементов из JavaScript позволяет реагировать на действия пользователя. Вы сможете менять стили элементов при наведении курсора, клике, фокусе и других событиях. Например, добавлять анимированный эффект при наведении курсора на кнопку, чтобы привлечь внимание пользователя.
• Освоите работу с классами элементов с помощью свойства classList. Сможете управлять видимостью элементов на странице, создавать сложные анимации и менять стили элементов в зависимости от действий пользователя.
• Научитесь получать и менять размеры элементов прямо из скрипта. Вы можете вычислять размеры элементов, определять их соотношение или находить расстояния между ними. Это может быть полезно при разработке интерактивных графиков, диаграмм, игр и других приложений.
• Более плотно поработаете с событиями и инструментом addEventListener. Сможете обрабатывать разные события пользователя. Например, отправку данных из формы, валидацию значений в инпутах. Научитесь создавать логику для скролла страницы.

С этими навыками вы сможете создавать привлекательные, удобные, конкурентоспособные сайты.

Но сначала, как всегда, предлагаем освежить знания из предыдущего модуля и пройти небольшой тест.

async function fetchUsers() {

try {
const res = await fetch('https://jsonplaceholder.typicode.com/users');

throw new Error('Ошибка!')
}
const data = await res.json();
if (data) {
data.forEach(elem => {
console.log(elem.name)
})
}
} catch (e) {
